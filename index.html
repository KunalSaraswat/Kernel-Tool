What is the Linux Kernel?

The Linux kernel is the heart of the Linux operating system, serving as the intermediary between hardware and user applications. It manages critical resources like CPU, memory, and I/O devices, enforces security policies, and provides a robust platform for running processes. Developed by Linus Torvalds in 1991, it‚Äôs now maintained by a global community and powers everything from smartphones to supercomputers.

$ uname -r

Displays the current kernel version, e.g., 6.9.5

$ lscpu | grep "Model name"

Shows CPU details managed by the kernel

The kernel operates in a monolithic yet modular architecture. All core functions (e.g., scheduling, file systems) run in kernel space for efficiency, but modules like drivers can be dynamically loaded or unloaded, enhancing flexibility. For example, loading a USB driver doesn‚Äôt require a kernel rebuild.

Key Features

  • Preemptive Multitasking: Enables efficient CPU time-sharing with the Completely Fair Scheduler (CFS).
  • Multi-user Support: Allows multiple users to run isolated sessions concurrently, ideal for servers.
  • Protected Memory: Ensures process isolation, preventing unauthorized memory access.
  • Virtual Memory: Uses paging and swapping to manage memory efficiently, supporting large address spaces.
  • Dynamic Modules: Supports loadable kernel modules (e.g., ext4 filesystem) for extensibility.
  • Symmetric Multiprocessing (SMP): Scales across multiple CPU cores for high performance.
  • Robust Networking: Implements advanced protocols like TCP/IP and netfilter for firewalls.

$ cat /proc/cpuinfo | grep "cpu cores"

Displays the number of CPU cores supported by SMP

Theoretical Foundations: Kernel Space vs User Space

User Space (Ring 3)

User space is the execution environment where all regular applications operate. It provides:

  • ‚Ä¢ Process Isolation: Each process runs in its own virtual address space
  • ‚Ä¢ Restricted Access: Cannot directly access hardware or kernel memory
  • ‚Ä¢ System Call Interface: Only way to request kernel services via well-defined APIs
  • ‚Ä¢ Context Switching: Requires mode transition (expensive operation)
  • ‚Ä¢ Memory Protection: Hardware-enforced via MMU and page tables

$ strace ls

Observe system calls made by user processes

Kernel Space (Ring 0)

The kernel space is the privileged core of the operating system with:

  • ‚Ä¢ Full Hardware Access: Direct control over CPU, memory, and devices
  • ‚Ä¢ Single Address Space: All kernel code/data shares one memory space
  • ‚Ä¢ Critical Operations: Manages interrupts, scheduling, memory allocation
  • ‚Ä¢ No Memory Protection: Any kernel code can access any kernel memory
  • ‚Ä¢ Preemption: Modern kernels support kernel preemption for responsiveness

$ cat /proc/kallsyms | head -20

View kernel symbol table (requires root)

Transition Mechanisms

User and kernel spaces communicate through carefully controlled interfaces:

System Calls

Software interrupts (int 0x80/syscall) that trigger mode switches. Linux has ~400 syscalls (x86_64).

Virtual Filesystem

/proc, /sys provide kernel data structures as files. Enables inspection without direct access.

Memory Mapping

mmap() allows controlled sharing of kernel buffers with user space (DMA, graphics).

Security Implications
  • ‚Ä¢ Kernel exploits are catastrophic (full system compromise)
  • ‚Ä¢ User processes are isolated (one crashing doesn't affect others)
  • ‚Ä¢ Modern protections: KASLR, SMAP, SMEP, SELinux

Kernel Space vs User Space

The Linux kernel operates in a privileged execution mode (Ring 0) with direct hardware access, while user applications run in Ring 3 with restricted privileges. This separation ensures system stability and security through hardware-enforced protection.

Key Differences:

  • ‚Ä¢ Memory Access: Kernel can access all memory; user space has virtual memory isolation
  • ‚Ä¢ Privileges: Kernel has CPU Ring 0; user space restricted to Ring 3
  • ‚Ä¢ Hardware: Direct hardware access vs. system call interface
  • ‚Ä¢ Context Switch: Mode switching requires CPU overhead (~100-1000 cycles)

$ cat /proc/meminfo | grep -E "(MemTotal|MemFree)"

View system memory allocation

$ sudo dmesg | grep -i "memory"

Check kernel memory management logs

Core Subsystems Interaction

  • System Call Interface: Provides controlled entry points (syscall table with 400+ entries) for user-kernel transitions.
  • Process Management: CFS scheduler with O(log n) complexity, load balancing across CPU cores, task_struct contains 1000+ fields.
  • Memory Management: 4-level page tables on x86_64, demand paging, copy-on-write, NUMA-aware allocation.
  • Virtual Filesystem: Unified API supporting 50+ filesystem types, dentries cache for path lookups.
  • Network Stack: Layer 3-7 protocol handling, socket buffers (sk_buff), traffic control subsystem.
  • Device Drivers: Character, block, network device classes with standardized interfaces.

System Call Execution Flow

System calls provide the only legitimate way for user programs to request kernel services. The process involves privilege escalation, parameter validation, service execution, and safe return to user mode.

Detailed System Call Process

  1. Library Call: User invokes read(fd, buf, count) from glibc, which wraps the actual system call.
  2. Syscall Instruction: CPU executes syscall instruction (x86_64) with syscall number in RAX register.
  3. Mode Switch: Hardware switches to Ring 0, saves user context (registers, stack pointer) on kernel stack.
  4. Parameter Validation: Kernel validates file descriptor, checks buffer accessibility using access_ok().
  5. Service Execution: Kernel executes sys_read(), may block for I/O, schedule other processes.
  6. Return: Results copied to user buffer, registers restored, CPU returns to Ring 3 user mode.
  7. Error Handling: Negative return values indicate errors, errno set by glibc wrapper.

Process Management

Process Fundamentals

A process in Linux is an executing program instance, managed by the kernel. Each process has its own address space, state, and resources. The kernel uses the task_struct data structure to track process details, forming a hierarchy starting with the init process (PID 1).

  • Process Control Block (PCB): task_struct stores PID, memory pointers, state, and context.
  • Process ID (PID): Unique identifier, e.g., systemd is typically PID 1.
  • Parent/Child Hierarchy: Processes are created by parents; init or systemd is the root.
  • States: Running, waiting (interruptible/uninterruptible), stopped, zombie, or dead.

Process Commands

$ ps -eF

Lists all processes with detailed info (PID, CPU, memory).

$ htop

Interactive process viewer with CPU/memory usage.

$ pstree -p

Shows process hierarchy with PIDs.

$ kill -SIGTERM 1234

Gracefully terminates process with PID 1234.

Process Hierarchy

Process Scheduler

The Linux kernel uses the Completely Fair Scheduler (CFS) to allocate CPU time fairly among processes. CFS tracks virtual runtime (vruntime) to ensure each process gets proportional CPU access. Real-time processes use priority-based scheduling (e.g., SCHED_FIFO).

  • CFS: Uses a red-black tree to prioritize processes with the lowest vruntime.
  • Real-Time Scheduling: SCHED_FIFO and SCHED_RR for time-critical tasks.
  • CPU Affinity: Binds processes to specific CPU cores for performance.

$ chrt -r -p 20 1234

Sets real-time SCHED_RR policy with priority 20 for PID 1234.

$ taskset -cp 0-3 1234

Binds PID 1234 to CPU cores 0-3.

Types of Processes

Linux processes are categorized based on their role and behavior. Each type serves distinct purposes in the system.

User Processes

Run in user space, initiated by users (e.g., firefox, bash). Subject to standard scheduling and permissions.

$ ps -u $USER

Lists processes owned by the current user.

Kernel Processes

Run in kernel space, handling system tasks (e.g., kworker, kswapd). Visible in ps with square brackets.

$ ps -ef | grep "\[k"\]

Lists kernel threads.

Daemon Processes

Background services (e.g., sshd, crond) that run continuously, often started by systemd.

$ systemctl list-units --type=service

Lists active daemons.

Real-Time Processes

High-priority processes for time-critical tasks (e.g., audio processing). Use SCHED_FIFO or SCHED_RR policies.

$ chrt -p $(pidof jackd)

Checks scheduling policy for the JACK audio server.

Process Life Cycle

A process transitions through various states during its lifetime, managed by the kernel. These states determine its execution status and resource allocation.

  • Created: Process is initialized via fork().
  • Running: Executing on CPU or ready to run.
  • Waiting: Blocked for I/O or events (interruptible or uninterruptible).
  • Stopped: Paused by signals (e.g., SIGSTOP).
  • Zombie: Terminated but awaiting parent to collect status.
  • Terminated: Process exits, resources are freed.

Process Creation

Processes are created using fork() to duplicate the parent and exec() to load a new program image. Threads, implemented as lightweight processes, share the same address space, managed via the clone() system call.

Process Creation Example

#include <stdio.h>
#include <unistd.h>
#include <sys/types.h>
#include <sys/wait.h>

int main() {
    pid_t pid = fork();
    if (pid == 0) { // Child process
        printf("Child PID: %d\n", getpid());
        execlp("/bin/ls", "ls", "-l", NULL);
        perror("execlp failed");
        return 1;
    } else if (pid > 0) { // Parent process
        printf("Parent PID: %d\n", getpid());
        wait(NULL); // Wait for child to complete
        printf("Child completed\n");
    } else {
        perror("fork failed");
        return 1;
    }
    return 0;
}

This program forks a child process that runs ls -l, while the parent waits for completion.

Interprocess Communication (IPC)

IPC mechanisms enable processes to exchange data or synchronize actions. Linux supports multiple methods tailored to different use cases.

Pipes

Unidirectional data channels for related processes (e.g., parent-child).

Fastest IPC, allowing processes to share a memory region.

Kernel Scheduling & Real-Time Kernels

Process Scheduling

The Linux kernel manages process execution using a scheduler to allocate CPU time. The Completely Fair Scheduler (CFS) is the default scheduler, prioritizing fairness and responsiveness.

Scheduling Policies

  • SCHED_NORMAL: Default for regular tasks (CFS)
  • SCHED_FIFO: First-In-First-Out for real-time tasks
  • SCHED_RR: Round-Robin for real-time tasks
  • SCHED_DEADLINE: Earliest Deadline First scheduling

CFS in Detail

CFS uses a red-black tree to track tasks, assigning CPU time based on virtual runtime (vruntime). Lower vruntime tasks get priority.

Key Concepts
  • Virtual runtime tracks CPU usage
  • Weighted fairness via nice values (-20 to 19)
  • Preemption for responsive multitasking
C Program Demonstrating Scheduling
#include "stdio.h"
#include "sched.h"
#include "unistd.h"
#include "sys/types.h"
#include "errno.h"

int main() {
    struct sched_param param;
    int policy;

    // Get current scheduling policy
    policy = sched_getscheduler(0);
    if (policy == -1) {
        perror("sched_getscheduler");
        return 1;
    }

    printf("Current policy: ");
    switch (policy) {
        case SCHED_OTHER: printf("SCHED_NORMAL\n"); break;
        case SCHED_FIFO: printf("SCHED_FIFO\n"); break;
        case SCHED_RR: printf("SCHED_RR\n"); break;
        default: printf("Unknown\n");
    }

    // Set SCHED_RR policy (requires root)
    param.sched_priority = 10;
    if (sched_setscheduler(0, SCHED_RR, ¶m) == -1) {
        perror("sched_setscheduler");
        return 1;
    }

    printf("Changed to SCHED_RR with priority %d\n", param.sched_priority);

    // Simulate work
    for (int i = 0; i < 5; i++) {
        printf("Working... %d\n", i);
        sleep(1);
    }

    return 0;
}

Real-Time Kernels

Real-time kernels prioritize deterministic response times for critical tasks, using PREEMPT_RT patches or dedicated real-time scheduling policies.

Real-Time Features

Real-Time in Practice

Real-time tasks can be programmed using SCHED_FIFO or SCHED_DEADLINE.

C Program for Real-Time Task
#include "stdio.h"
#include "sched.h"
#include "time.h"
#include 'unistd.h"

#define NSEC_PER_SEC 1000000000L

// Simple nanosecond sleep
void nsleep(long nsec) {
    struct timespec ts = { .tv_sec = 0, .tv_nsec = nsec };
    nanosleep(&ts, NULL);
}

int main() {
    struct sched_param param = { .sched_priority = 99 };

    // Set SCHED_FIFO policy (requires root)
    if (sched_setscheduler(0, SCHED_FIFO, ¶m) == -1) {
        perror("sched_setscheduler");
        return 1;
    }

    printf("Running as SCHED_FIFO with priority %d\n", param.sched_priority);

    // Lock memory to avoid swapping
    if (mlockall(MCL_CURRENT | MCL_FUTURE) == -1) {
        perror("mlockall");
        return 1;
    }

    // Simulate periodic real-time task
    for (int i = 0; i < 10; i++) {
        printf("Real-time task iteration %d\n", i);
        nsleep(100 * 1000000); // 100ms
    }

    munlockall();
    return 0;
}

Advanced Scheduling Topics

SCHED_DEADLINE Example

#include "stdio.h"
#include "sched.h"
#include "linux/sched.h"
#include "unistd.h"

int main() {
    struct sched_attr {
        __u32 size;
        __u32 sched_policy;
        __u64 sched_flags;
        __s32 sched_nice;
        __u32 sched_priority;
        __u64 sched_runtime;
        __u64 sched_deadline;
        __u64 sched_period;
    } attr = {
        .size = sizeof(struct sched_attr),
        .sched_policy = SCHED_DEADLINE,
        .sched_runtime = 10 * 1000 * 1000,  // 10ms
        .sched_deadline = 20 * 1000 * 1000, // 20ms
        .sched_period = 20 * 1000 * 1000,   // 20ms
    };

    // Set SCHED_DEADLINE (requires root)
    if (syscall(SYS_sched_setattr, 0, &attr, 0) == -1) {
        perror("sched_setattr");
        return 1;
    }

    printf("Running as SCHED_DEADLINE\n");

    // Simulate periodic task
    for (int i = 0; i < 10; i++) {
        printf("Deadline task iteration %d\n", i);
        usleep(10000); // 10ms
    }

    return 0;
}

C Program Demonstrating Paging Effects

                        
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

#define PAGE_SIZE 4096  // Typical page size
#define NUM_PAGES 10

int main() {
    char *pages[NUM_PAGES];

    // Allocate pages
    for (int i = 0; i < NUM_PAGES; i++) {
        pages[i] = malloc(PAGE_SIZE);
        if (!pages[i]) {
            perror("malloc failed");
            exit(1);
        }
        printf("Allocated page %d at %p\n", i, pages[i]);
    }

    // Write to each page to ensure they're mapped
    for (int i = 0; i < NUM_PAGES; i++) {
        pages[i][0] = i;  // Touch first byte of each page
    }

    // Pause to examine memory maps
    printf("Check /proc/%d/maps now. Press enter to continue...", getpid());
    getchar();

    // Free pages
    for (int i = 0; i < NUM_PAGES; i++) {
        free(pages[i]);
    }

    return 0;
}

C Program Showing Segmentation Effects

                        
#include <stdio.h>
#include <stdlib.h>

// Global variables (stored in data segment)
int global_var = 10;
const int global_const = 20;

void print_addresses() {
    // Local variables (stored in stack segment)
    int local_var = 30;
    static int static_var = 40;  // Stored in data segment

    printf("Function address: %p\n", &print_addresses);
    printf("Global variable: %p\n", &global_var);
    printf("Global constant: %p\n", &global_const);
    printf("Static variable: %p\n", &static_var);
    printf("Local variable: %p\n", &local_var);

    // Dynamically allocated memory (heap segment)
    int *heap_var = malloc(sizeof(int));
    printf("Heap variable: %p\n", heap_var);
    free(heap_var);
}

int main() {
    printf("Main function: %p\n", &main);
    print_addresses();

    // Show segment differences
    printf("\nTypical memory layout:\n");
    printf("1. Text segment (code): %p\n", &main);
    printf("2. Data segment (globals): %p\n", &global_var);
    printf("3. Heap segment (grows up): %p\n", malloc(1));
    printf("4. Stack segment (grows down): %p\n", &global_var);

    return 0;
}

Memory Allocation Code Example

                    
#include <linux/module.h>
#include <linux/kernel.h>
#include <linux/init.h>
#include <linux/slab.h>

static int __init mem_init(void) {
    int *ptr;

    // Allocate memory using different methods
    ptr = kmalloc(sizeof(int) * 100, GFP_KERNEL);
    if (!ptr) {
        printk(KERN_ALERT "kmalloc failed\n");
        return -ENOMEM;
    }
    printk(KERN_INFO "kmalloc allocated at %px\n", ptr);
    kfree(ptr);

    // vmalloc for large, virtually contiguous memory
    ptr = vmalloc(PAGE_SIZE * 16);  // 16 pages
    if (!ptr) {
        printk(KERN_ALERT "vmalloc failed\n");
        return -ENOMEM;
    }
    printk(KERN_INFO "vmalloc allocated at %px\n", ptr);
    vfree(ptr);

    // Page allocator example
    struct page *page = alloc_pages(GFP_KERNEL, 0);  // Allocate 1 page
    if (!page) {
        printk(KERN_ALERT "alloc_pages failed\n");
        return -ENOMEM;
    }
    printk(KERN_INFO "Allocated page at %px\n", page_address(page));
    __free_pages(page, 0);

    return 0;
}

static void __exit mem_exit(void) {
    printk(KERN_INFO "Memory module unloaded\n");
}

Memory Mapping and Advanced Topics


                

                    
Memory Mapping Example

                    

                        
#include <stdio.h>
#include <stdlib.h>
#include <sys/mman.h>
#include <fcntl.h>
#include <unistd.h>

int main() {
    const size_t length = 4096;  // One page
    const char *file_path = "/tmp/mmap_example";

    // Create a file for mapping
    int fd = open(file_path, O_RDWR | O_CREAT, 0666);
    if (fd == -1) {
        perror("open failed");
        exit(1);
    }

    // Extend file to needed size
    if (ftruncate(fd, length) == -1) {
        perror("ftruncate failed");
        close(fd);
        exit(1);
    }

    // Map the file into memory
    char *mapped = mmap(NULL, length, PROT_READ | PROT_WRITE,
                       MAP_SHARED, fd, 0);
    if (mapped == MAP_FAILED) {
        perror("mmap failed");
        close(fd);
        exit(1);
    }

    printf("File mapped at address: %p\n", mapped);

    // Use the memory
    sprintf(mapped, "Hello from memory mapping!");
    printf("Content: %s\n", mapped);

    // Clean up
    if (munmap(mapped, length) == -1) {
        perror("munmap failed");
    }
    close(fd);

    return 0;
}

Page Table Walk Example

                    

                        This example shows how to examine page table entries (requires root privileges):
                    

                    

                        
#include <stdio.h>
#include <stdlib.h>
#include <stdint.h>
#include <inttypes.h>
#include <sys/mman.h>

// For page table walking (Linux specific)
#include <sys/types.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>

void print_pagemap_info(uint64_t value) {
    printf("Page table entry: 0x%" PRIx64 "\n", value);
    printf("  Present: %d\n", (value & (1ULL << 63)) ? 1 : 0);
    printf("  Swapped: %d\n", (value & (1ULL << 62)) ? 1 : 0);
    printf("  File/Shared: %d\n", (value & (1ULL << 61)) ? 1 : 0);
    printf("  Soft-dirty: %d\n", (value & (1ULL << 55)) ? 1 : 0);
    printf("  Exclusive: %d\n", (value & (1ULL << 56)) ? 1 : 0);
}

int main() {
    int fd;
    uint64_t value;
    void *addr = malloc(4096);  // Allocate a page

    if (!addr) {
        perror("malloc failed");
        return 1;
    }

    // Open pagemap file for the current process
    fd = open("/proc/self/pagemap", O_RDONLY);
    if (fd == -1) {
        perror("open pagemap failed");
        free(addr);
        return 1;
    }

    // Calculate offset in pagemap file
    uintptr_t virtual_addr = (uintptr_t)addr;
    size_t offset = (virtual_addr / 4096) * sizeof(value);

    if (pread(fd, &value, sizeof(value), offset) != sizeof(value)) {
        perror("pread failed");
        close(fd);
        free(addr);
        return 1;
    }

    print_pagemap_info(value);

    close(fd);
    free(addr);
    return 0;
}

#include <linux/uaccess.h>

#define DEVICE_NAME "simple_fs"
#define BUF_LEN 1024

static int major;

static int device_open(struct inode *inode, struct file *file) {
    printk(KERN_INFO "Device opened\\n");
    return 0;
}

static ssize_t device_read(struct file *filp, char *buffer, size_t length, loff_t *offset) {
    int bytes_read = 0;
    if (*offset == 0) {
        bytes_read = copy_to_user(buffer, msg, length);
        *offset += length;
    }
    return bytes_read;
}

static ssize_t device_write(struct file *filp, const char *buf, size_t len, loff_t *off) {
    copy_from_user(msg, buf, len);
    return len;
}

static struct file_operations fops = {
    .read = device_read,
    .write = device_write,
    .open = device_open,

    printk(KERN_INFO "Registered with major number %d\\n", major);
    return 0;
}

static void __exit fs_exit(void) {

// simple_char_driver.c
#include <linux/module.h>
#include <linux/fs.h>
#include <linux/uaccess.h>

#define DEVICE_NAME "mychardev"
#define BUF_LEN 1024

static int major;
static char msg[BUF_LEN];
static int msg_len = 0;

// Called when the device is opened
static int dev_open(struct inode *inodep, struct file *filep) {
    printk(KERN_INFO "[mychardev] Device opened\n");
    return 0;
}

// Called when data is read from user space
static ssize_t dev_read(struct file *filep, char *buffer, size_t len, loff_t *offset) {
    if (*offset >= msg_len) return 0;
    int bytes_read = copy_to_user(buffer, msg, msg_len);
    *offset += msg_len;
    return msg_len;
}

// Called when data is written to the device
static ssize_t dev_write(struct file *filep, const char *buffer, size_t len, loff_t *offset) {
    msg_len = (len < BUF_LEN) ? len : BUF_LEN;
    copy_from_user(msg, buffer, msg_len);
    return msg_len;
}

// Define file operations
static struct file_operations fops = {
    .open = dev_open,
    .read = dev_read,
    .write = dev_write,
};

// Called when module is loaded
static int __init driver_init(void) {
    major = register_chrdev(0, DEVICE_NAME, &fops);
    if (major < 0) {
        printk(KERN_ALERT "[mychardev] Failed to register device\n");
        return major;
    }
    printk(KERN_INFO "[mychardev] Registered with major number %d\n", major);
    return 0;
}

// Called when module is removed
static void __exit driver_exit(void) {
    unregister_chrdev(major, DEVICE_NAME);
    printk(KERN_INFO "[mychardev] Unregistered device\n");
}

module_init(driver_init);
module_exit(driver_exit);
MODULE_LICENSE("GPL");
MODULE_AUTHOR("Kunal Saraswat");
MODULE_DESCRIPTION("A Simple Linux Char Driver");

Socket Programming (Kernel)

                

                    Sockets enable communication between devices or processes over a network. While commonly used in user space, they can also be used inside the Linux kernel for special use cases like in-kernel proxies or monitoring.
                


                


// kernel_socket.c - Create TCP socket in kernel space
#include <linux/module.h>
#include <linux/net.h>
#include <linux/in.h>
#include <net/sock.h>

static struct socket *sock;

static int __init sock_init(void) {
    struct sockaddr_in addr = {
        .sin_family = AF_INET,
        .sin_port = htons(9000),
        .sin_addr.s_addr = htonl(INADDR_LOOPBACK),
    };

    // Create a TCP socket in kernel space
    if (sock_create_kern(&init_net, AF_INET, SOCK_STREAM, IPPROTO_TCP, &sock) < 0) {
        printk(KERN_ALERT "[ksocket] Socket creation failed\\n");
        return -1;
    }

    // Attempt to connect to the loopback address
    if (sock->ops->connect(sock, (struct sockaddr *)&addr, sizeof(addr), 0) < 0) {
        printk(KERN_ALERT "[ksocket] Connection failed\\n");
        sock_release(sock);
        return -1;
    }

    printk(KERN_INFO "[ksocket] Successfully connected\\n");
    return 0;
}

static void __exit sock_exit(void) {
    if (sock) {
        sock_release(sock);
        printk(KERN_INFO "[ksocket] Socket released\\n");
    }
}

module_init(sock_init);
module_exit(sock_exit);
MODULE_LICENSE("GPL");
MODULE_AUTHOR("Kunal Saraswat");
MODULE_DESCRIPTION("Kernel TCP Socket Example");

Netfilter Example: Packet Filtering

            

                Netfilter allows kernel modules to intercept, modify, or drop packets at various stages of the networking pipeline.
            


            


// netfilter_drop_icmp.c
#include <linux/module.h>
#include <linux/netfilter.h>
#include <linux/netfilter_ipv4.h>

static struct nf_hook_ops nfho;

// Drop ICMP packets
static unsigned int hook_func(void *priv, struct sk_buff *skb,
                             const struct nf_hook_state *state) {
    struct iphdr *iph = ip_hdr(skb);
    if (iph && iph->protocol == IPPROTO_ICMP) {
        printk(KERN_INFO "[netfilter] Dropping ICMP packet\\n");
        return NF_DROP;
    }
    return NF_ACCEPT;
}

static int __init nf_init(void) {
    nfho.hook = hook_func;
    nfho.hooknum = NF_INET_PRE_ROUTING;
    nfho.pf = PF_INET;
    nfho.priority = NF_IP_PRI_FIRST;

    nf_register_net_hook(&init_net, &nfho);
    printk(KERN_INFO "[netfilter] Hook registered\\n");
    return 0;
}

static void __exit nf_exit(void) {
    nf_unregister_net_hook(&init_net, &nfho);
    printk(KERN_INFO "[netfilter] Hook unregistered\\n");
}

module_init(nf_init);
module_exit(nf_exit);
MODULE_LICENSE("GPL");


                
This module blocks all ping (ICMP) traffic at the pre-routing hook.

    printk(KERN_INFO "Goodbye, Kernel World!\n");
}

module_init(hello_init);
module_exit(hello_exit);

#include <linux/fs.h>
#include <linux/cdev.h>

static int device_open(struct inode *, struct file *);
static int device_release(struct inode *, struct file *);
static ssize_t device_read(struct file *, char __user *, size_t, loff_t *);
static ssize_t device_write(struct file *, const char __user *, size_t, loff_t *);
static long device_ioctl(struct file *, unsigned int, unsigned long);

static struct file_operations fops = {
    .open = device_open,
    .release = device_release,
    .read = device_read,
    .write = device_write,
    .unlocked_ioctl = device_ioctl
};

static int __init chardev_init(void) {
    register_chrdev(MAJOR_NUM, DEVICE_NAME, &fops);
    // ...
}
